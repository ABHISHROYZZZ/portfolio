Project Overview

This project involved developing a feature store for machine learning models to enable consistent feature engineering across multiple models. The solution centralized feature definitions, computation, and serving, reducing inconsistencies and improving model performance for a tech startup with 15+ ML models in production.

Challenges

  • Inconsistent feature definitions across different models
  • Redundant feature computation leading to inefficiencies
  • Lack of feature versioning and lineage tracking
  • Difficulty in reproducing model training environments

Solution & Implementation

We implemented a feature store using Feast as the core framework, backed by Redis for online feature serving and BigQuery for offline storage. Developed custom connectors for data sources and established automated pipelines for feature computation and validation.

Technology Stack

Feast Redis BigQuery Python Kubernetes Airflow
